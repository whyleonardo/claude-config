Review the currently staged changes for quality and correctness.

!`git diff --staged`

## Review Checklist

Use the `reviewing-code` skill for detailed guidelines.

Focus on:
1. **Correctness** - Does it work as intended?
2. **Type Safety** - Are types properly defined?
3. **Error Handling** - Are errors handled appropriately?
4. **Tests** - Are changes tested?
5. **Style** - Does it follow project conventions?

Provide:
- Summary of changes
- Issues found (if any)
- Suggestions for improvement
- Verdict: Ready to commit or needs changes
